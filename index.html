SOFTWARE ENGINEER

Terminal Application

As part of the WMS software, I was responsible for synchronizing data with the ERP system using an XML ACL layer. This involved configuring and setting up XML files to ensure seamless data integration. I created generic XML generator generate xml only by given object to the mehanism.


Alerts Module

I developed this module to notify supervisors via email and other system users of important events. It also included a safety alarm feature for the warehouse, which automatically halts all robotic operations to allow for maintenance or troubleshooting when someone enters the warehouse.


RoboticsConnector Module

I developed the entire module, which included translating locations from a preconfigured warehouse map, automatically refreshing robot locations using SignalR for the frontend, and integrating RabbitMQ to read these locations. Additionally, the module was responsible for creating various transport missions in the robots' external mission software. This module also included unit and integration tests.


ErpConnector Module

I developed the part of this module responsible for validating customer data. My implementation used the strategy design pattern to handle data validation, ensuring that unwanted or duplicate data (which could be sent by customers via SQL procedures) did not enter the WMS.


PLC Message Broker

To manage the frequent calls (every second) from PLC devices to RabbitMQ, I implemented an outbox pattern. This ensured that duplicate messages were not processed by our WMS, maintaining data integrity.
